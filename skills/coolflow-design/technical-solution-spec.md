# 前端技术方案规范（coolflow-design 技能内化版）

> 本文件是 `coolflow-design` 技能的**内化规范**（自包含），由团队原始技术方案规范内化优化而来：
> 移除了原文的目录/命名规则（统一由技能的 coolflow 落点规则接管）与威胁性措辞，
> 输入来源改为「用户输入 + clarify 阶段的需求澄清结果」。
>
> design 技能据此生成技术方案，**严格遵守下列规则与模板**。

## 输入（下文统称 X）
X = 用户命令行输入 + `coolflow/<需求名称>/clarify-<日期>.md` 的「需求澄清结果」部分。
其中可能包含：`$需求名称$`、`$Tapd$`、`【相关文档】`、`$涉及页面$`、`$改动内容$`、`$生成要求$` 等字段。

## 生成规则
- **不生成实现代码，只生成前端技术方案。**
- 严格按下文「方案模板」生成，章节与顺序不得增删。
- 模板中两个 `**` 之间的内容是**生成要求**，不要呈现在最终结果中。
- 不明确的地方，通过对话询问用户（或回到 clarify 阶段补澄清）。
- 输出落点与文件名由 design 技能的「阶段产物落点规则」决定（`coolflow/<需求名称>/design-<日期>.md`），
  本规范不再自带命名/目录规则。

#### 2.2、流程视角
**
1、分析 X 中 $生成要求$，获取用户指定的图类型；**若 $生成要求$ 未指定图类型，默认使用泳道图**。
    - 若为泳道图（含未指定而默认的情况），用 PlantUML 生成（用 "|" 模拟泳道）；其他图类型用 Mermaid。
2、分析 X 中 $涉及页面$ 和 $改动内容$，以页面为维度，为每个页面每个场景分别生成改动后的图表：
    - 找到项目中对应页面/模块，分析总结主要逻辑
    - 找到与改动点相关的内容
    - 先画改动前的图表（尽量详细，重点体现改动内容在改动前的逻辑）
    - 再以改动前图表为基础画改动后的图表（尽量详细，重点体现改动后的逻辑）
3、改动后的图中用橙色高亮标注本次改动：
    - Mermaid：用 rect 语法 + 颜色高亮
    - PlantUML：用 "<<#Orange>>" 放到对应节点行末 ";" 后高亮
**

#### 2.3、架构视角
**
- 分析项目结构与 X 中改动内容，用 Mermaid 生成前端架构图，着重体现本次改动。具体要求：
    - 仅输出一个 mermaid 代码块（flowchart），不要任何解释或多余文字。
    - 布局 flowchart TB；模块用 subgraph 可嵌套；整体协调美观、避免大面积留白（同级模块尽量等长等宽）：
        - 顶层模块为项目，多个顶层模块水平排列
        - 次级模块为页面或公共组件模块，水平排列，公共组件在左、页面在右
        - 底层模块为具体方法或逻辑，竖直排列
    - 形状：全部矩形
    - 连线：严禁任何箭头/连线（-->、-.->、-->|label| 及任何连接符号）
    - 样式：classDef 定义三种：default(白底/#111 边框)、changed(#F7C0C0/#C0392B)、added(#C7F3C7/#2E7D32)
    - 初始化：mermaid init 设 theme=base、字体 Arial、禁用阴影
    - 必须可直接在 mermaid live editor 渲染
**
